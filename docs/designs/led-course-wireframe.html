从首页到跑通的体验路径

把北极星指标(完成首课率 > 60%)的瓶颈画出来:高职学生第一次进来 → 走完 5 步 → 看到 LED 闪。这个页面聚焦路径上的 3 个关键决策点,不是完整的视觉稿。

01 — 现状问题

首课路径上三个会让学生走丢的坑

PRD v2 §6.5 / §14 已经把"5 步走完 LED 课"定成验收硬指标,但当前实现里这三个点会让"走完"变成"走半路就关"。

01

编辑器锁死

CoursePlayer 里 CodeEditor disabled={true},学生只能看不能改。PRD §6.5 写的是 "taskCode 预填",意思是"先填一个,你照着改/调试",不是"锁死只能看"。

02

首屏没有引导

现在 Editor 默认代码是 "D5 Demo blink D13",但首课 LED 闪烁的入口藏在哪?首页没看过,但 PRD §14 的"未登录裸跑已实现"这条没人推到首页。0 引导 = 0 转化。

03

画布硬编码色

CanvasPanel 用了 #1a2028 网格 / #ff5252 红线 / #0079d1 蓝框,跟 daisyUI 主题没关系。切暗色/亮色主题时画布不会跟着变,主题分裂。

02 — 用户路径

从打开浏览器到第 5 步 LED 闪

每一站都是流失点。下面这条路径上 UI 投入的优先级 = 路径上对应页面的优先级。

04 — 课程 step 改稿

把"看代码"变成"跟代码"

课程播放器的核心矛盾:学生该不该自己写?下面是两种走法,推荐 A。

推荐
A · 代码可改,任务驱动

编辑器可写,左上角贴"任务卡"(例:"把 delay 改成 200")。学生写对了,任务卡变绿,解锁下一步。

  • 写代码 = 主动学习,留存高
  • 失败能本地重试,省 AI 配额
  • 任务卡是 P1 hint 字段的天然落点
  • 需要课程 step schema 加 check 字段(目前是 title/context/taskCode/taskWiring 4 字段)
不推荐
B · 代码只读,Run 就行

保持 disabled。学生在 step 5 点 Run 看 LED 闪就算"完成"。

  • 实现简单,不改 schema
  • 但学生没"自己写"的参与感,完课率上不去
  • 北极星指标 完成首课率 > 60% 大概率达不到

05 — 5 步内容大纲

从看懂到跑通,每步一个动作

课程文案你(项目主理人)来写,UI 这边要配合的是"每步只问学生做一件事"。下面是 UI 视角的 5 步节奏建议。

1
认识 LED
看懂电路符号,知道哪条腿长是正极。
动作: 只读。看图,不动手。
2
让 LED 亮
补全 setup() 里的 pinMode。
动作: 改一行代码,Run 看到 LED 亮。
3
让 LED 闪
补全 loop(),加 delay。
动作: 改 delay 数值,Run 看到闪。
4
调速度
理解 delay 是节奏。
动作: 把 1000 改 100,看加速。
5
完成 + 留资
看到完整闪烁,引导保存/分享。
动作: 点 Run 完整跑通,完课态。
