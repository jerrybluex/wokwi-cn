Design System · D11 · 2026-07-02 · v0.1 草案

教学产品的视觉语言

这一页是 wokwi UI 的设计基础。daisyUI 当底层(避免造轮子),我们在它上面加一层品牌 token。三个关键词:温度、清晰、统一。

01 — 设计原则

做这个,不做什么

教学产品的 UI 跟工具型产品不一样。学生看 5 秒就要知道"下一步干啥",看 30 秒就要"我点这个会出错吗"。下面的原则是从 PRD §2 用户(小李 / 小张,高职学生)倒推出来的。

要做

  • 主按钮永远只有 1 个(主操作)
  • 状态色:绿 / 橙 / 红 / 蓝,不要紫色 AI 渐变
  • 教学文案:动词 + 对象("把 delay 改成 200")
  • 画布配色独立 token 化,跟 daisyUI 主题同源
  • 间距用 4 的倍数,跟 daisyUI scale 对齐

不做

  • emoji 当 icon(替换 lucide-react)
  • 渐变 / 阴影 / 动效堆砌
  • 深色模式硬切(画布本身是暗色,UI 是浅色,统一)
  • 暗色 / 浅色 / 蓝色三主题切换(MVP 范围外)
  • WCAG AA 全部达标(MVP 不要求,但颜色对比 ≥ 4.5:1)

02 — 配色

主色暖橙,辅助深绿

暖橙 #c97b3f 是教学温度感(不是工具的冷蓝),深绿 #2f6f5e 是"完成 / 正确"的正向反馈。语义色按 daisyUI token 走,只是把 active 状态调一下。

03 — 字体

06 — 组件模式

在 daisyUI 上面叠一层品牌

直接用 daisyUI 的 `btn` / `card` / `input`,通过 Tailwind 配置注入品牌 token。下面是 MVP 用到的关键组件的对照。

BUTTON

规则:每屏只有一个 btn-primary;次按钮用 btn-ghost;危险操作(删除)用 btn-error 或透明带 hover 红。

BADGE
已完成 ✓ 已分享 P1 · 路上 未验证 现线上

规则:badge 只用来标状态(已完成 / 共享 / 验证),不承载操作入口(那是 button 的事)。

CARD
LED 闪烁
5 步,从打开 IDE 到 LED 闪
P1 · 路上
更多课程
按钮 / 电位器 / 串口……

规则:白色 card + 10px 圆角 + 1px 描边 + sm 阴影。占位用 dashed + 斜纹。

ALERT
已保存 14:32 · 云端
编译错误: 'pinMode' was not declared
你的项目还没有保存,先点右上角 💾

规则:alert 必须包含 icon(替代 emoji),文案动词开头。error 红、success 绿、info 蓝,不用 warn 黄当 warning。

INPUT

规则:focus ring 用 primary-soft(8% alpha),不用 daisyUI 默认的厚 ring。disabled 用 muted 文字。

07 — 元件库视觉规范

8 件统一规则

现在每个元件自己画 SVG,视觉风格脱节(LED 圆、UNO 长方形、Button 圆按钮)。统一规则:PCB 蓝深色板 + 金属 pin + 白色标签 + 暖橙 #c97b3f 强调(选中态)。

视觉规则 (8 件通用)

  • PCB 蓝 #1a4a8a (UNO) / 深灰 #262c37 (其他元件主体) — 不要纯黑
  • Pin 灰 #888 引线 + #1a2028 圆点 (暗背景) / #5a6173 圆点 (亮背景)
  • 标签字 JetBrains Mono 9-10px,白色 #d8dee9
  • 选中态 外框 1.5px primary-soft 虚线 + 内描边 primary 1px
  • 连线态 pin 圆点变 primary #ff5252,wire 跟随线 primary 1.5px dashed
  • 运行态 (LED / Button 等):LED 亮起 #ff5252 主色 + glow,Button 按下 #c97b3f 主色

08 — 画布主题感知

把硬编码色换成 CSS 变量

CanvasPanel 现在用了 6+ 个硬编码色 (#1a2028 / #ff5252 / #0079d1 / #ffb300),跟 daisyUI 主题脱节。改成 CSS 变量,让画布跟着 UI 主题走(将来加 dark mode 也不破)。

--canvas-bg · #1a1d24 --canvas-grid · #262a33 --canvas-grid-major --canvas-wire · #e07a5f --canvas-select · #5b9dd9 --canvas-pin-active 
// apps/web/src/index.css (扩展)
:root {
  /* UI (daisyUI theme) */
  --primary: #c97b3f;
  --bg: #f7f5f0;
  --paper: #ffffff;

  /* Canvas (独立 token,默认暗色) */
  --canvas-bg: #1a1d24;
  --canvas-grid: #262a33;
  --canvas-grid-major: #34394a;
  --canvas-wire: #e07a5f;
  --canvas-pin: #5a6173;
  --canvas-pin-active: #ffb300;
  --canvas-select: #5b9dd9;
  --canvas-text: #d8dee9;
  --canvas-board: #1a4a8a;
}

// CanvasPanel.tsx 改 svg fill/stroke 引用 var(--canvas-xxx)
// 而不是硬编码 #1a2028 / #ff5252

09 — 动效

教学产品要"轻盈",不要"炫技"

MVP 阶段只用 3 类动效:状态切换(150ms)、抽屉/抽屉化(200ms)、LED 闪烁(500ms 周期)。其他交给浏览器默认。

过渡时长

state (hover/focus)
150ms
.15s ease

200ms
.2s ease-out

500ms
sync w/ code

Easing

--ease-out: cubic-bezier(0.22, 1, 0.36, 1);
--ease-in-out: cubic-bezier(0.65, 0, 0.35, 1);

/* 默认走 ease-out,不用 linear */
.btn, .card, .alert {
  transition: all var(--ease-out) 150ms;
}
