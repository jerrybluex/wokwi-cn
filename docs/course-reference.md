# 高职单片机课程参考 · 调研笔记

> 目的：搞清楚国内高职单片机课程到底在教什么、用什么平台、学时怎么分配。我们做的产品必须能"嵌进"现有教学，而不是另起炉灶。

**调研时间**:2026-07-01
**方法**:搜索 + 阅读 5 份高职 / 中职课程标准 + 1 本人民邮电出版社规划教材目录
**结论摘要**:高职主流是 **MCS-51 单片机(8051 内核)**,不是 Arduino。详见后文。

## 3. 教学法(几乎所有标准都强调)

- **项目驱动 / 任务驱动**:用"汽车转弯灯""交通灯""数字电压表""防盗报警器"等具象任务承载知识点
- **理论实践一体化**:教室和实验室合一,边讲边做
- **形成性评价 + 终结性评价**:平时 40% + 课设 30% + 期末 30% 是典型比例
- **考核形式**:上机考试 + 实操 + 报告

---

## 4. 使用的工具与平台

| 类别 | 主流选择 |
|---|---|
| **单片机** | **MCS-51(8051 内核)**:AT89C51 / STC89C52 / CC2530 |
| **仿真软件** | **Keil C51**(代码) + **Proteus / ISIS**(电路仿真) |
| **编程语言** | **C51**(C 语言的 51 扩展),少部分用汇编 |
| **开发板** | 学校自配 51 实验箱 |

**重要发现**:国内高职 **不是 Arduino**。Keil + Proteus 是绝对主流。

> 这是个战略问题:我们的 PoC 是 Arduino,真正的高职市场是 51。

这本书是 21 世纪高职高专规划教材,作为内容参考很有价值。

**亮点**:
- 校企联合打造
- 强调"虚拟接口与虚拟器件"思想(正好契合我们的产品!)
- 配套 PPT、源代码、芯片手册、STC 寄存器定义

**对我们的启发**:项目结构、章节顺序可以直接借鉴。我们做的不只是仿真器,是**课程内容的载体**。

## 6. 课程标准里的"考核形式"(直接抄)

典型期末上机考试结构(标准 C):

```
考试时间 90 分钟,上机开卷可携带教材

一、文件夹规范(10 分)
二、Keil 软件使用熟练度(10 分)
三、程序功能正确性(50 分)
四、代码缩进格式(10 分)
五、烧录工具使用(10 分)
六、试验现象正确(10 分)

题目样例:
01 按键控制 LED 流水灯。S4 控制方向,第一次按下从上往下流,
   第二次按下从下往上流。要求使用 T0 定时,LED 每 0.5s 移动一个。

02 数码管动态显示。分两次显示 8 位学号,先前四位,1s 后再后四位。
   T0 定时。

03 数码管秒表 00-59 循环,2 个 LED 指示,T0 定时。
```

→ 这意味着 **判题系统** 要能检查:
- 文件夹 / 项目结构
- 代码编译通过
- 引脚行为(D0-D7 在 0.5s 时刻改变状态)
- 串口 / 显示输出是否符合预期

## 8. 关键启示(给我们的产品决策)

1. **如果定位高职 → 必须支持 MCS-51**,而不是只看 Arduino。
   - 51 仿真器可以基于 SDCC + iverilog 实现,也可以基于现有的 51 模拟器(如 MCU 8051 IDE 的开源版本)。
   - simavr 是 AVR 仿真器,不能直接复用。
2. **学时的"前 30 学时"是黄金入口**,所有学生都从这里开始,流失率最高。
   - 这是我们产品最该发力的地方:让一个学生在 30 分钟内完成 LED 闪烁从 0 到能。
3. **判题要硬约束**:期末考试的形式很结构化,我们的判题引擎要能直接对接这套题面。
4. **学生考核关心"文件夹 / 缩进 / 项目结构"** —— 这意味着我们的平台要原生是"项目"概念,不是单文件编辑器。
5. **教师备课要的是"项目库"**,不是"课程视频"——我们的课程编辑器要让老师能直接 fork 一个现成项目改电路图、改代码,然后布置下去。

## 11. 高职里的 Arduino 路径(补充调研)

虽然 51 是高职主流,但 Arduino 在高职有 3 个细分场景:

### 11.1 学术支撑

**《基于 Arduino 的高职物联网相关课程改革设计》** — 南宁职业技术学院,2015 年
- 核心观点:Arduino 引入物联网课程能**降低实验室成本 + 降低学生入门门槛**
- 适合做:物联网应用技术专业的"专业基础课"
- 这是我们产品定位的最强学术背书

### 11.2 课程内容来源(直接候选)

**直接复用价值**:

### 11.4 课程编辑器的步骤结构(产品决策)

基于"5 模块 / 18 任务 / 8 环节",我们课程编辑器的 Step Schema 应该是 8 字段:

```typescript
interface CourseStep {
  // 第一组:学生输入
  goal: string;          // 学习目标 ("能描述 LED 的工作原理")
  context: string;       // 情境导入 ("你刚入职车联网公司,接到...")
  // 第二组:题目
  taskCode: string;      // 任务初始代码
  taskWiring: Wiring;    // 任务初始电路(SVG / JSON)
  // 第三组:辅助
  hint1: string;         // 第一层提示(给方向)
  hint2: string;         // 第二层提示(给关键字)
  hint3: string;         // 第三层提示(给完整方案)
  // 第四组:验收
  check: CheckFunction;  // 评价反馈(自动判题)
}
```

8 个字段完整覆盖了"行云模式",同时也兼容更简单的"描述 / 任务 / 检查点" 3 字段模式(默认填充)。
