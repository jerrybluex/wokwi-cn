## 2. PRD 未明示但产品上的缺口

### 2.1 ✅ 编辑器裸跑已支持(无需登录可仿真)

`apps/web/src/pages/Editor.tsx` 不在 `<RequireAuth>` 包装下,**未登录用户可以直接进编辑器 + 写代码 + Run + 看 LED 闪**。`onSaveNow` 在没有 projectId 时尝试 `projectsApi.create`,会因缺 JWT 失败 —— 等于软门槛。这跟 PRD §10 风险表里"看课程无需注册,只保存要"一致。**无需修改**。

### 2.2 ✅ 编辑器已有 Stop / 重置按钮

`onRun` + `onStop` + `↺ 重置` 按钮齐全,学生写死循环可救回。**之前我口头说"学生跑飞了怎么救"是误判,这里订正**。

### 2.3 ❌ 反馈入口缺失

PRD §9 成功指标 / §10 风险表都没提"4 周验证期里怎么收集用户反馈"。Sentry 抓 crash,但学生"卡住但没崩"的声音接不到。

**建议**:**P0**,上线前必须补。最简实现:页脚/侧栏一个"反馈"按钮 → `mailto:` 或外跳到腾讯问卷/金数据/Notion form。

### 2.4 ⚠️ 示例项目模板只有 1 个(Demo)

`apps/web/src/pages/Editor.tsx` 有 `📦 Demo` 按钮,加载 "UNO + 220Ω + LED" 默认电路。**只有 1 个**。

学生第一次进来空白编辑器一脸懵 —— 即使有 Demo 按钮,他们不知道可以点。

**建议**:**P0**,把 Demo 扩展成模板库,至少 3 个(LED 闪烁 / 按钮读取引脚 / 电位器调亮度)。点击直接载入到编辑器。

### 2.5 ❌ 分享页缺"再创作"按钮

`apps/web/src/pages/ShareView.tsx` 是纯只读页,**没有"用这个模板"按钮**。

看到别人作品 fork 到自己账户 → 现在的流程是复制代码、手动建项目,体验割裂。

**建议**:**P0**,加一个 `🚀 用这个模板` 按钮 → 调后端 API,基于 shareId 项目复制成当前用户新项目。需要登录态。

### 2.6 ⚠️ 仿真器走的是 JS 软仿真(降级路径)

PRD §10 风险预案里写 "D2 收工时 LED 不闪 → 切 PoC 的 JS 软仿真" —— 这个降级路径**已被触发**。代码里 `apps/web/src/sim/runner.ts` 是纯 JS 软仿真,没有 simavr WASM 集成。MVP 验收里"LED 闪 / 串口出文本"还是过的,但**仿真精度跟真实 AVR 不一致**,长期是个技术债。

**建议**:**P2(Phase 2 / 3 再评估)**。MVP 接受软仿真,但要在 PRD §6.1 加一条说明"MVP 走 JS 软仿真,精度为 best-effort;Phase 2 评估是否换 simavr WASM"。

**决策 7 详情**(PRD §14 AI 响应阈值调整):

- **签字**:产品经理(2026-07-02)
- **原阈值**:AI 响应 < 5s(PRD v2.1)
- **新阈值**:AI 响应 **P95 < 6s**
- **理由**:
  - chat 模型 P95 实测 5.1s,接近 5s 阈值;长输出(error 类多段)需要 buffer
  - v4-pro 推理模型 P95 9.2s 远超阈值,Phase 2 评估(stream 优化 / 阈值放宽 / 提示工程)
  - 阈值从单一 "< 5s" 调整为 "P95 < 6s",对齐行业标准(MDN / Google SRE 推荐 P95 < X 而非 P50)
- **验证数据**:general 三轮对比 30 次调用
  - v4-pro(10 次):P50 6.2s / P95 9.2s / 最慢 11.7s
  - chat(400 tokens,10 次):P50 2.4s / P95 5.2s / 最慢 6.0s
  - chat(600 tokens,10 次):P50 2.5s / P95 5.1s / 最慢 5.3s
- **B 路径实测失败**:max_tokens=600 微改善,P95 几乎持平,未解决 P95 <5s 问题
- **AiCall 表可查**:`apps/server/prisma/dev.db` 30 条记录(taskType + tokensIn/Out + latency)
- **PRD §13 ADR-002 不动**:DeepSeek V3 为主仍成立;v4-pro 升级尝试是 Phase 2 议题

**决策 8 详情**(CoursePlayer check 字段 MVP 落地):

- **签字**:产品经理(2026-07-02,UIer 接活 D11 余下 + D12-D15)
- **原计划**:check 字段 Phase 2(PRD §6.5 标注"后期可加",与"课程编辑器"捆绑)
- **调整**:MVP 落地,跟"课程编辑器"解绑
- **理由**:完课率是 PRD §9 唯一北极星指标,>60% 才算 MVP 成功。check 字段是任务卡驱动的关键(student-loops,不是 editor-loops)
- **schema**:Step interface 加可选字段 `check?: { kind: "api-used" | "pattern"; value: string; label?: string }`,**不破坏现有 4 字段**(title / context / taskCode / taskWiring)
- **判定**:轻量(词法 + 关键 API 调用模式匹配),不严格 AST
- **UI 改造**:
  - step title 升级"任务卡"(例:"把 delay 改成 200")
  - 完成自动变绿 + 解锁下一步
  - 任务驱动 student-loop,完课率有望 >60%
- **工程量**:**+1 天**(UIer 接,跟 D11-D15 一起做)
- **与"课程编辑器"区别**:
  - check 字段 = MVP 学生侧(student-loop)
  - 课程编辑器 = Phase 2 教师侧(teacher-loop)
  - 两者独立,check 提前不破坏"课程编辑器 Phase 2 才做"决策
- **不动 PRD §6.5 主结构**(仍是 4 字段,check 是可选扩展)
