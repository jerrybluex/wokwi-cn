## 1. 速览

**一句话**:让中国学生**打开浏览器就能学单片机**,写代码、看到 LED 在闪、跟着课程一步步走,有 AI 助教随时答疑,不需要硬件。

**项目类型**:B2C + B2B2C SaaS Web 应用
**项目名**:`wokwi`(暂定,后续可改)
**产品形态**:浏览器即用、无需下载、自动保存、可分享作品
**核心差异化**:**AI 助教 + 中文课程库 + 学生社区**

## 2. 目标用户

### 2.1 主用户:高校 / 高职学生

| 画像 | 描述 |
|---|---|
| 年级 | 本科大二-大三 / 高职 |
| 专业 | 电子信息、自动化、物联网、计算机 |
| 痛点 | 没买板子 / 实验室排队 / 写完代码看不到效果 |
| 场景 | 课前预习、课上练习、考前复习、课后作业 |

**典型用户小李**:
> 大二电子信息工程,宿舍没 Arduino。学校实验室一三五开放,周二晚上想预习下周"中断与定时器"那节课,写了代码没法验证,只能看视频。这是他想用的工具。

### 2.2 次用户:单片机课教师

- 一线授课,1 学期带 30-300 人
- 痛点:批改实操作业难、看不到每个学生进度

### 2.3 付费方(候选)
- 学校(整校采购)
- 教辅机构(课时包)
- 企业培训(内训定制)

## 3. 用户痛点

| 优先级 | 痛点 | 现状 |
|---|---|---|
| **P0** | 硬件门槛高 | 没板子的学生压根无法练习 |
| **P0** | 反馈循环太长 | 写代码→烧录→接线→看效果,5-15 分钟一次 |
| **P0** | 同学之间难分享 | 实物操作没法"发链接给你看" |

| 角色 | 一句话价值 |

## 5. 范围

### 5.1 In Scope(做)

**Phase 1 — MVP(单人 10-12 周,周均 35-40 工时)**
- Web 仿真器(接 simavr WASM)
- 元件库:12 个常用器件(以 Arduino UNO 为主控)
- 学生端:代码编辑器、串口监视器、电路编辑、项目保存/分享
- 课程播放器:步骤式引导、解锁机制、提示
- **AI 助教**:基于 LLM 的代码问答、错误解释、引导式提示
- 用户系统(邮箱 + 密码)
- 一键分享 / Fork 课程
- 内容:5 门课程 / 18 任务 / 32 学时,**骨架直接对接《Arduino 编程控制与应用》**(行云新能 / 机械工业出版社,2023)

**Phase 2 — 教学闭环(单人 12-14 周)**
- 教师判题系统(I/O 断言 / 串口断言 / 波形断言)
- 班级 + 作业 + 进度看板
- 课程市场(其他教师贡献)
- 手机适配 + PWA
- **进阶课程**:中断 / 定时器 / I2C 传感器

**Phase 3 — 商业化 + 51 内核(单人 16+ 周,或找合伙人)**
- 学校 SSO(CAS / OAuth / 微信扫码)
- 私有化部署(Docker 包)
- 企业培训定制
- **MCS-51 内核支持**(高职主流市场):基于 SDCC + 8051 模拟器
- 元件库扩展至 51 体系(LED / 数码管 / LCD1602 / 矩阵键盘 / A/D / D/A)

### 5.2 Out of Scope(明确不做)

- ❌ 自定义元件编辑器(由我们提供)
- ❌ 多人实时协作
- ❌ 移动原生 App
- ❌ 跨设备实时同步(单设备内自动保存足够)

**核心 8 个**:

1. **作为学生**,我想不下载任何东西就在浏览器写 Arduino 代码

3. **作为学生**,我想跟着步骤学习,卡点能看提示
4. **作为学生**,我想保存项目,下次回来继续
5. **作为学生**,我想分享作品给同学看(只读链接)
6. **作为学生**,我想 Fork 别人的作品做二次创作
7. **作为学生**,我想在手机上查看我的进度
8. **作为学生**,我想给老师看我的代码,被点评

## 7. 功能需求

### 7.1 仿真引擎
- 接 `@wokwi/simavr-js`(simavr 的 WASM 端口)
- 内核:ATmega328P(UNO/Nano)
- 引脚事件精度:1 ms 级别
- 元件与仿真器解耦,所有元件以"模型 + 视图"两层实现

### 7.2 元件库(Phase 1,12 个)

| 类别 | 元件 |
|---|---|
| 主控 | Arduino UNO、Arduino Nano |
| 输入 | 按钮、电位器、光敏电阻、超声波 HC-SR04 |
| 输出 | LED(红/黄/绿/蓝)、RGB LED、舵机 SG90、蜂鸣器、七段数码管 |
| 通信 | OLED (SSD1306)、MPU6050 |
| 基础 | 电阻、电容 |

每个元件需要:**真实行为 + 视觉外观 + 元件库 JSON 描述**(供第三方课程引用)。

### 7.3 代码编辑器
- 语法高亮(CodeMirror 6 或 Monaco)
- 自动补全(Arduino API)
- 编译错误分行内提示
- 内置代码片段模板
- 支持中英文混排注释

### 7.4 课程播放器

**步骤结构:8 字段(基于行云新能教材的"8 环节教学法")**

| 字段 | 含义 | 来源 |
|---|---|---|
| `goal` | 学习目标 | "理解 LED 的点亮原理" |
| `context` | 情境导入 | "你刚入职车联网公司..." |
| `taskCode` | 任务初始代码 | Arduino 模板 |
| `taskWiring` | 任务初始电路 | SVG / JSON |
| `hint1` | 第一层提示 | 给方向 |
| `hint2` | 第二层提示 | 给关键字 |
| `hint3` | 第三层提示 | 给完整方案 |
| `check` | 自动判题 | 引脚断言 / 串口断言 / 波形 |

UI 上学生只看到 `context → task → check` 三段,但内部数据 8 字段都给,适配高职"任务驱动"教学法。

**其他**:
- 解锁机制:完成上一步才能下一步
- 一键复制课程为个人副本
- 进度持久化(关联到用户)
- 内容骨架直接借鉴《Arduino 编程控制与应用》(行云新能 / 机械工业出版社,2023)18 个任务,**详见 [`./docs/course-reference.md`](./docs/course-reference.md) §11.2**

### 7.5 AI 助教
- 入口:编辑器右侧抽屉 + 课程页悬浮按钮
- 能力 v1:
  - **代码问答**:解释某段代码做了什么 / 报错原因
  - **错误翻译**:把编译器错误翻成大白话,附修改建议
  - **引导式提示**:学生卡住时给 3 层递进提示(给方向 → 给关键字 → 给完整方案)
  - **元件推荐**:根据任务描述推荐电路应该加什么
- 实现:**DeepSeek V3 为主**(中文 + 低价),复杂代码 / 长上下文升级到 **Claude Sonnet**
- 走 SSE 流式输出,后端加 API Key 代理层(不直接暴露学生端)
- 安全:学生代码不存云端,只在请求时附给 LLM;提供"清空对话"按钮
- 限额:免费用户每日 20 次问答,Pro 用户无限(为 Phase 3 商业化埋点)
- 不做 v1:语音、AI 自动评分、AI 生成实验

### 7.6 用户系统
- 邮箱 + 密码注册
- 找回密码(邮件验证)
- 角色:学生(Phase 1)/ 教师(Phase 2)
- 单设备会话足够

## 10. 关键风险

| 风险 | 概率 | 影响 | 缓解策略 |
|---|---|---|---|
| simavr WASM 体积大、首次加载慢 | 高 | 中 | 懒加载 + Service Worker 缓存 + 国内 CDN |
| 老师不买账(习惯 Proteus / Multisim) | 中 | 高 | 先做学生侧,教师案例靠口碑 |
| 元件仿真与实板行为不一致 | 中 | 高 | 元件回归测试套件 + 与实板对比清单 |
| 学校采购流程长(国标 / 招投标) | 高 | 中 | 优先小 B(培训机构、工作室),大 B 慢慢来 |
| 独立开发者带宽不足 | 高 | 高 | 一次只推一条线,功能严格分阶段 |
| 国内外法律合规(simavr 许可证) | 低 | 中 | 用 Wokwi 开的 Apache 2.0 端口,合规 |

**真实节奏提示**(单人开发者的常见踩坑):
- 仿真器集成永远比预期长 1.5 倍(WASM 编译、内存、兼容性)
- 课程内容是"看不见的工作",5 门课实际写起来要 2 周
- 内测至少留 1 周,学生用起来和开发者视角差很大
- 不要并行做 AI 助教 + 仿真器 + 内容,挑一个先做透

## 12. 假设与不做的事

**假设**(可证伪):
- 用户主要使用 Chrome 桌面浏览器
- 国内高校学生群体对纯英文界面有显著阻力
- simavr WASM 可合法在国内使用(已确认 Apache 2.0)
- 仿真响应 < 100ms 足以让"打字-反馈"感觉即时

**默认选择**(可以反驳):
1. 先做学生侧,教师侧放 Phase 2
2. 先 SaaS,不做私有化部署
3. 优先中文界面,英文 V2 再考虑
4. 仿真器内核先 ATmega328P 一种,后续加 ESP32

### 13.1 开源策略

- **仿真器内核**:`@wokwi/simavr-js` 已经是 Apache 2.0,我们基于它的二次封装也开源
- **元件模型**:12 个常用元件的"模型 + 视图"代码开源(MIT)
- **平台代码**(用户系统、课程播放器、AI 助教代理):闭源
- **课程内容**:可以开源也可以闭源,看创作者意愿
- **理由**:仿真器开源 = 招聘信号 + 社区贡献(学生加新元件);平台闭源 = 商业化保护

## 14. 验收标准(Phase 1 完成的判定)

满足以下全部条件视为 MVP 完成:

- [ ] 12 个元件全部仿真行为正确,与实板对比通过测试清单
- [ ] 浏览器打开 < 3 秒可交互
- [ ] 仿真响应延迟 P95 < 100 ms(本地测)
- [ ] 邮箱注册 → 写代码 → 保存 → 分享链接全流程跑通
- [ ] **AI 助教** 可用:能解释报错、回答"为什么 LED 不亮"、给 3 层提示
- [ ] **5 门课程 / 18 任务**全部基于行云新能教材骨架,可被新用户走通完成
- [ ] 至少 5 个真实学生用户完整走完一遍无严重阻塞
- [ ] Lighthouse 性能分 > 70
- [ ] 关键页面在 Chrome / Safari / Edge 都能用
- [ ] 单元测试覆盖率 > 50%(仿真引擎核心)
- [ ] 有简单监控(Sentry 之类)接住线上错误
