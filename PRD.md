# PRD v2 — wokwi

> 在浏览器里学单片机的中文教学产品。**本版本聚焦 MVP** —— 2 周冲刺快速出活。Phase 2 / 3 内容不在此文档,需要时另起 v3。

## 1. 速览 (Vision)

让中国高职/本科学生**打开浏览器就能学单片机**,代码→ LED 闪→ 串口输出,有 AI 助教随时答疑,不需要硬件。

**当前 Sprint 目标(2 周内)**:
1. 学生能跑通真实仿真(LED 闪烁、串口打印)
2. AI 助教能解决学生最常见问题
3. 注册 → 写代码 → 保存 → 分享链路通

**后续**:MVP 验证后,再设计 Phase 2(教师 / 判题 / 班级 / 商业化)。

**内容素材定位**:课程是 hand-crafted,**教材(包括行云新能、人邮、教科院等)只作为参考来源**,我们的 step schema 跟教材不强对齐。

---

## 2. 目标用户 (Users)

**Primary**:中国高职 / 本科大二大三学生(电子、自动化、物联网、计算机相关)
**Secondary**:中学生创客、自学入门的工程师

**典型用户**:
- **小李**:大二电子信息,宿舍没 Arduino;想预习下周单片机课的"中断",写了代码没设备跑
- **小张**:大三物联网,临近毕业设计,想用 Arduino 攒个物联网方案,但实验室预约不到
- **王老师**:高职单片机任课教师,想给学生一份课前测,但没资源搭仿真平台(这个群体在 MVP 不服务,记一下)

MVP 阶段只服务 **小李 / 小张** 这一类学生,王老师放 Phase 2。

## 4. MVP 范围 (In/Out)

### 4.1 In Scope(2 周内)

| 功能 | 验收 |
|---|---|
| 邮箱注册 / 登录 / 找回密码 | 全流程能跑 |
| 项目保存到云端 | 自动 10 秒一次,至少 100 个项目安全 |
| 仿真引擎(@wokwi/simavr-js WASM) | LED 闪、串口输出 demo 通 |
| 12 个元件 | UNO/LED/按钮/电位器/电阻/七段/舵机/蜂鸣器/OLED/MPU6050/超声波/RGB LED |
| 代码编辑器(CodeMirror 6) | Arduino C 高亮 + 错误捕获 |
| 串口监视器 | 1s 内更新,过滤 ANSI |
| 电路画布(SVG) | 拖放 + 连线 + 撤销 / 重做 |
| 1 门示范课(LED 闪烁,5 step) | 学生能从头走完不出错 |
| AI 助教(DeepSeek V3) | 代码问答 / 错误翻译 / 引导提示 |
| 分享链接(只读) | 任何人点开可见 |

### 4.2 Out of Scope(本 MVP 不做)

- ❌ 教师账号 / 班级 / 作业 / 进度看板
- ❌ 自动判题系统(I/O 断言等)
- ❌ 课程编辑器(只做学生 playback,不做老师用)
- ❌ 多课程库(1 门足够验证)
- ❌ MCS-51 内核 / ESP32
- ❌ PWA / 移动 App / Electron
- ❌ 商业模式 / 订阅 / 付费
- ❌ 私域部署 / SSO / 学校采购
- ❌ AI 助教语音 / 自动评分 / AI 自动生成课
- ❌ 离线模式 / 同步冲突处理
- ❌ 第三方登录(微信/GitHub)
- ❌ 用户侧代码导入(只能新建项目)

**为什么砍这些**:2 周时间只够走完核心 student-loop。任何"看起来有用但不阻塞 MVP"的都砍。

## 5. 用户故事 (MVP 6 个)

1. 作为学生,**不下载任何东西**,浏览器打开就写 Arduino 代码
2. 作为学生,**写完代码立刻看到 LED 闪、串口输出**
3. 作为学生,**跟着 5 step 走完 led 课**,无致命阻塞
4. 作为学生,**问 AI 助教"为什么 LED 不亮"**,30 s 内拿到解释
5. 作为学生,**邮箱注册 + 保存项目**,关掉再开还能找到
6. 作为学生,**发只读链接给同学**,同学打开能看到代码 + 当前电路

## 6. 功能需求 (MVP 必须)

### 6.1 仿真引擎

- `@wokwi/simavr-js`(Apache 2.0,JS 浏览器版本)
- 内核:ATmega328P(UNO 内核)
- 引脚事件精度:1 ms tick
- 元件与仿真器解耦:`元件模型` + `元件视图` 双层

### 6.2 元件库(MVP 12 个)

主控:Arduino UNO
输入:按钮、电位器、超声波 HC-SR04
输出:LED(多色)、RGB LED、舵机 SG90、蜂鸣器、有源、七段数码管
通信:OLED(SSD1306)、MPU6050
基础:电阻

每个元件提供:模型(JS class)+ 视图(SVG 节点)+ 引脚定义(JSON)+ 单元测试

### 6.3 代码编辑器

- CodeMirror 6 + Arduino 模式(自定义 C++ grammar,关键字高亮)
- 自动补全:Arduino API(pinMode/digitalWrite/delay 等,Phase 1.v1)
- 编译错误行内提示
- 编译:走 WASM 内的 avr-gcc,或云端 API(选便宜方案)

### 6.4 电路画布

- SVG 渲染,每元件一个 `<g>` 节点
- 拖放:左侧元件卡 → 画布
- 连线:点 pinA → 半透明跟随线 → 点 pinB 完成
- 撤销 / 重做(MVP 用命令栈,实现简化版)
- 删除选中元件 / Esc 取消连线中
- Pan & zoom(简单实现,MVP 不要求完美)

### 6.5 课程播放器

**Step 数据模型(MVP 用 4 字段)**:

```typescript
interface Step {
  title: string;
  context: string;        // markdown 渲染
  taskCode: string;       // 预填的 Arduino 代码
  taskWiring: WiringJSON; // 预填电路 JSON
  // 后期可加: hint1-3 / check
}
```

UI:左文 + 右编辑器/画布,点下一步 / 上一步。
进度:localStorage + 登录后同步云端(进度仅记录完成 step,不做题对错)。

### 6.6 AI 助教

- 入口:编辑器右上角抽屉图标
- 3 类能力(MVP):
  - 代码问答:"这段代码做什么"
  - 错误翻译:编译报错 → 大白话 + 修复建议
  - 引导提示:卡住时给方向(只给提示,不给答案)
- 模型:DeepSeek V3(中文好、价格低)
- 安全:学生代码不存,请求后丢弃
- 限额:**每账号每日 20 次**(免费)
- 实现:后端 `/api/ai/chat` 代理(后端拿 API Key),前端 SSE 流式输出

### 6.7 用户系统

- 邮箱 + 密码注册(Argon2id hash)
- 邮件验证(24h token)
- 找回密码(邮件 token)
- JWT session(HttpOnly cookie,7d 过期)
- 单设备 session(不强制下线其他)

### 6.8 项目保存

- 自动保存:每 10 秒一次,只在有变化时提交
- 单账号多项目(project list + 单项目详情 API)
- 分享链接:`GET /p/{share_id}`,只读 JSON 返回项目快照

## 7. 非功能需求 (MVP)

| 维度 | 要求 |
|---|---|
| 性能 | 仿真响应 P95 < 100 ms |

| 监控 | Sentry(前端 + 后端) |
| 日志 | 结构化 JSON,12 周保留 |

## 10. 关键风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| simavr WASM 集成卡 > 2 天 | 中 | 高 | Day 1 立即开干;Day 2 没通就 fallback 到 Wokwi 自己写的 JS 软仿真(已经在我们的 PoC 里) |
| 单人 2 周扛不住 | 高 | 高 | 砍功能 > 砍质量。学生 MVP 用 8 元件也够 demo;砍 OLED + MPU6050 + RGB LED + 七段 + 舵机 + 超声波 → 留下 6 件 |
| DeepSeek API 调用挂 | 低 | 中 | 缓存常见回答;失败给降级文案让用户继续;复杂切 Claude |
| 注册转化低(看了就跑) | 中 | 中 | 看课程无需注册(只保存要);分享链接无需登录 |
| 服务器被攻击 / 数据泄漏 | 低 | 高 | MVP 阶段小流量;Argon2id + simple rate limit |

## 12. 假设

- 单人开发,周 35-40 h 投入,2 周是 MVP 交付期

## 14. 验收标准 (MVP 完成判定)

满足以下全部条件 → MVP 通过:

**功能性**
- [ ] 邮箱注册 → 验证 → 登录 全流程通
- [ ] 至少 8 元件完整功能(LED、按钮、电位器、电阻、UNO、超声波、舵机、蜂鸣器即可)
- [ ] 编辑代码 → Run → LED 闪 / 串口出文本
- [ ] 画布拖放 + 连线 + 撤销
- [ ] 1 门示范课(LED 闪烁 5 step),用户能走通
- [ ] AI 助教能回答"为什么 LED 不亮",响应 < 5 s
- [ ] 分享链接可访问、只读

**质量**
- [ ] Lighthouse 性能分 > 70
- [ ] P95 加载 < 3 s,WASM 不阻塞首屏
- [ ] P95 仿真 tick < 100 ms
- [ ] 关键页面 Chrome / Safari / Edge 都能用
- [ ] 单元测试(仿真器核心)覆盖 > 50%

**上线**
- [ ] 公网 https(临时域名可)
- [ ] Sentry 监控有数据
- [ ] 至少 1 个非自己人端到端试用过 30 min 无致命阻塞
